# Evaluation harness

This directory houses the **code** for the objective measurement layer:
datasets, runners, metrics, visualisation. All **artifacts** they produce
(JSON baselines, PNG figures, Markdown reports) land under `outputs/` —
see [`outputs/README.md`](../outputs/README.md).

```
eval/
├── datasets/            # downloaders + GT parsers (WIDER FACE, LFW)
├── rosbag_converters/   # turn images / videos into rosbag2 mcap streams
├── runners/             # end-to-end evaluators (run_detection_eval.py, …)
├── metrics/             # IoU, AP, recognition accuracy, latency — unit tested
├── reports/             # generate_report.py — builds Markdown from outputs/baselines/
└── visualize.py         # matplotlib figures into outputs/figures/

outputs/                 # ← everything generated by the above ends up here
├── baselines/           # versioned JSON snapshots
├── figures/             # PNG visualisations
└── reports/REPORT.md    # integrated markdown report with figures embedded
```

## Quick start

### 1. Get WIDER FACE val

```bash
mkdir -p ~/datasets/wider_face && cd ~/datasets/wider_face
wget https://huggingface.co/datasets/CUHK-CSE/wider_face/resolve/main/data/wider_face_split.zip
wget https://huggingface.co/datasets/CUHK-CSE/wider_face/resolve/main/data/WIDER_val.zip
unzip wider_face_split.zip
unzip WIDER_val.zip
```

That leaves you with `~/datasets/wider_face/WIDER_val/images/...` (3,226 images)
and `~/datasets/wider_face/wider_face_split/wider_face_val_bbx_gt.txt`.

### 2. Run the current baseline

```bash
# From the package root
source /opt/ros/humble/setup.bash
make eval-detection-baseline
```

This invokes `python3 -m eval.runners.run_detection_eval --backend dlib_hog`
over the full val set (~10–15 min on a single core) and drops the result in
`outputs/baselines/v0_dlib_hog_wider_val.json`.

### 3. Compare runs

```bash
make eval-report             # outputs/reports/REPORT.md
make visualize               # outputs/figures/*/*.png
make outputs                 # everything above in one shot
```

## Detection runner reference

```bash
python3 -m eval.runners.run_detection_eval \
    --backend {dlib_hog,dlib_cnn,yunet,mediapipe} \
    --output eval/baselines/<label>.json \
    [--model path/to/weights.onnx] \
    [--limit N] \
    [--score-threshold 0.5] \
    [--label "human-readable label"]
```

Result JSON schema is stable so regression diffs between commits are
straightforward — see `eval/reports/generate_report.py` for the columns.

## Rosbag converter

Used to feed any image set into a ROS 2 pipeline as `sensor_msgs/Image`.

```bash
python3 -m eval.rosbag_converters.images_to_rosbag \
    --wider-val-root ~/datasets/wider_face \
    --output /tmp/wider_val.bag \
    --fps 5.0 \
    --limit 100
```

The bag bundles two topics:
* `/camera/color/image_raw` — the image with a synthetic monotonic stamp
* `/eval/image_id` — `std_msgs/String` carrying the dataset image_id, so
  the runner can correlate predictions with ground truth deterministically.

## Adding a new backend

1. Implement a class in `eval/runners/run_detection_eval.py` with `name` attribute
   and `detect(bgr) -> list[(Box, score)]`.
2. Register it in `BACKENDS = {...}`.
3. Run the smoke fixture under `test/unit/test_detection_runner.py` to
   confirm the interface contract.
4. Run on WIDER FACE val — the JSON lands in `outputs/baselines/`.
5. `make visualize-detection && make eval-report` to refresh figures and
   `outputs/reports/REPORT.md`.
